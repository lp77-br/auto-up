TERMS OF USE AND PRIVACY POLICY

Last updated: June 15, 2026

1. Introduction

These Terms of Use and Privacy Policy govern the use of the AutoUp bot, developed and maintained by LP77. By adding and configuring the AutoUp bot on your Discord server, the server administrator fully agrees to the guidelines for data processing, collection, and use established in this document.

2. Data Collection

AutoUp operates under the strict principle of data minimization, storing only the information vital to the operation and security of the P2P partnership network. The data processed and stored in our infrastructure includes:

  • Platform Identifiers: Server ID (Guild ID), Owner ID, and the ID of the text channel designated for continuous integration.
  • Authorized Content: The descriptive promotional text manually configured and submitted by the administrator.
  • Network Telemetry: Invite links generated exclusively by the bot's internal engine and quantitative traffic statistics (total member count and usage increments tied to the official link).

Scope Restriction: AutoUp does not collect, read, listen to, or retain the content of messages sent by users, presence data, chat history, unrelated audit logs, or any personally identifiable information (PII) of individual members.

3. Use of Information

The collected data is used exclusively for the following systemic purposes:

  • To feed the asynchronous automation cycle of the P2P engine, distributing your server's promotional message to the queue of partner communities.
  • To intercept new user entry events (Gateway Events) to calculate the conversion rate and validate the integrity of cached invites.
  • To process quantitative reach metrics, network node status, and generate internal usage rankings for the bot's commands.

4. Security and Sharing

The JSON configuration structure of the servers is kept in an isolated environment and processed in memory cache (RAM) to mitigate disk exposure (I/O). The administration (LP77) does not sell, assign, rent, or share the collected metadata with third-party entities under any circumstances.

5. Data Retention and Deletion

Control and jurisdiction over the provided data belong to the server owner.

  • Temporary Inactivation: The administrator can halt the partnership flow at any time by revoking the active channel via bot commands.
  • Definitive Revocation: Kicking or removing the bot from the server immediately ceases the collection of new data and marks the node as inactive in the global network. For the physical expurgation of residual metadata in the application's database, the administrator holding the Owner ID may open a direct request via the support channel.

6. Compliance with the Discord API

The software operates in mandatory compliance with the Discord API Terms of Service and Developer Guidelines. The Server Members Privileged Intent is strictly consumed for the differential calculation of invite usage (fetchInvites) upon a member's entry, with no persistent tracking, profiling, or construction of external user databases beyond the scope of partnership automation.

7. Contact and Support

For vulnerability reports, inquiries about data compliance, or technical deletion requests, server maintainers must exclusively report to the official AutoUp protocol support server on Discord.
